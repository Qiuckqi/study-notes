#**一、 linux**
稳定、免费、网络功能丰富、，并且非常安全

需要做一个虚拟裸机电脑：vmware、virtualbox

## **1. 创建虚拟机**

1. 兼容性采用默认
2. 选择稍后安装操作系统 方便观察步骤
3. 选择linux操作系统 选择对应版本
4. 虚拟机装在vm文件夹 表示虚拟机 
   虚拟硬盘装在dk文件夹 为巡硬盘文件目录
5. 处理器数量不要超过实际物理处理器规格
   内存不能低于640MB 一般不要低于700mb
6. 选择创建虚拟磁盘 一般10G即可 不选择*立即分配所有磁盘空间*
7. 根据自身实际的磁盘类型选择是将虚拟磁盘否拆分成多个文件
8. 在光驱中选择ISO文件，开始安装操作系统

## **2. 安装操作系统**
1. 直接选择第一项
2. 选择不检测硬件，若检测硬件一来时间花费较多，二来检测完后会直接退出， 需要重新进入

> **交换分区**： 在硬件技术不发达的时候，将硬盘的一部分空间当作内存使用，主要用于存储内存中使用较少的数据，一般设为内存的两倍。

# **分区原理**

##windows的分区原理
1. 挂载：	操作系统目录与硬盘分区建立联系的过程
2. 挂载点： 	被挂载的操作系统目录即为挂载点
3. 挂载类型：自动、手动。windows的挂载类型都是自动的。
4. 根目录：	每个盘即为根目录
5. 文件占据占据空间，各自挂载点目录下文件占据对应挂载点本身的磁盘空间

##**Linux**
1. 挂载与挂载点：	同windows
2. 挂载类型：		**自动和手动。两种都有。**
	    自动：系统安装创建的挂载点，后期自动与硬盘分区建立联系
		手动：系统运行中，临时添加的U盘、移动硬盘不会被系统应用起来，需要手动创建文件陆慕并使其与硬件进行挂载。
3. 根目录：		**只有一个 “/”**
4. 文件占据分区空间，非挂载点文件目录向上寻找**最近挂载点**，占据该挂载点对应分区空间。
5. 与新硬件形成挂载

		①把原挂载点目录内部的旧文件释放出去
		②再进行挂载操作



#根目录
目录

1. **/bin**目录：即binary 二进制

		“指令”对应“执行程序文件”目录

2. **/sbin**目录:super binary  超级二进制
		
		目录中位root用户可执行的指令

3. ***/usr**目录： nuix system resource 用于安装各种软件*

		软件安装完毕形成的对应指令存放于此
		/usr/bin  
		/usr/sbin 

4. ***/dev**目录： 系统硬件设备目录  *

5. ***/home**目录：用户的存放目录*

		创建普通用户，会在该目录下创建对应的同名home目录，用户进入系统时首先进入home目录，并拥有绝对的最高权限。

6. **/root**目录


7. **/proc**目录： 内存映射目录，


8. ***/var**目录：variable 可变、易变*
	
		存储经常会发生变动的文件（增加修改删除）
		常用于部署项目程序文件
9. **/boot**目录：用于存储系统启动，存放系统启动核心文件


10. ***/etc**目录：系统主要配置文件目录*

11. **/lib**目录： 系统资源文件类库目录
12. **/selinux**目录：secure enhanced linux 安全增强型linux
		
		对系统形成保护，干扰系统安装软件

#三. 权限操作
###1. 权限
权限划分为读写执行 ：	Read Write	eXecute
针对用户的划分为主人、同组用户、其他用户：
在ls -l中，文件权限在系统中的体现：User	Group	Other
		
		-rwxr-xr-x 2 Jiangsu Nanjing
		- ：文件类型。 -为文件，d为目录
		rwx： 主人权限
		r-x:  同组用户权限
		r-x:  其他组用户文件
		Jiangsu： 文件的主人
		Nanjing： 文件的组别

###2. 权限的设置

①每个单元的权限只能使用一个+或-

②可以同时对多个组设置权限，使用逗号隔开

③权限可以为rwx的一个或者多个

		chomd 指令
		1）针对单个组别设置权限，成为相对方式
		chmod u+rwx	filename		//给文件的主人增加读写执权限
		chmod g-rx	filename		//给文件的同组用户删除读执权限
		chomd u+/-rwx,g+/-rwx,o+/-rwx filename		//用户之间用逗号","割开，"/"在此表示或者

		chmod +/-rwx filenam		//无视具体组别 对所有用户设置权限

###3. 权限的使用
		###对文件而言
		读：查看文件内容
		写：修改文件内容（但可以强制修改）
		执行：执行二进制文件/批处理文件或shell脚本文件


		###对目录而言
		读：查看目录内部文件信息的权限
		写：是否可在该目录下创建删除文件该目录
		执行： 是否可以进入该目录
		
###4. 只对文件的强制操作
在没有r或w权限时，使用！可强制写入保存

		对文件有w无r，强制写保存，新入内容会覆盖文件原内容
		对文件没有w,可以强制写保存
		非主任用户对文件无w：
		  ①该文件上级目录针对该修改者没有开放w，则不成功
		  ②该文件上机目录针对该修改者开放了w,则成功
		修改成功后文件的主人组别改为修改者的主人组别

#七、软链接和硬链接

##1、软连接 link
类同windows系统的快捷方式  
作用：可对硬盘空间进行合理分配

具体设置：
>ln -s 源文件 软链接文件

**使用注意**：  
若软连接文件与源文件不在同一级目录，则源文件需要设置为绝对路径，否则其无效  
软链接本身存储着源文件的路径数据，占据一定大小
普通文件和目录都可以设置软连接  
源文件被删除，对应软连接变为无效链接；若再创建同名链接，则软连接恢复为有效链接文件

##2、硬链接  
系统中文件的名称（引用）即为硬链接  
给文件增加名称的过程就是创建“硬链接”  
一个文件有多个名字，都是指向同一个文件实体的硬链接
  
>ln [-d] 源文件 硬链接

一个文件若有多个名称，即使其中一个名称被删除，也不影响其他名称的使用（即文件实体依然存在）。
若文件只有一个名称，执行rm删除指令，仅仅是删除文件名称。
系统垃圾回收机制会定期检查该文件是否存在名称（引用），名称不存在的就会被回收。

###**注意**：  
①设置硬链接并不需要绝对路径  
②只有普通文件可以设置，目录不能设置硬链接  
③同一个源文件的所有硬链接文件必须**在同一个硬盘、同一个分区**里


目录的硬链接数一般为3  
本身是一个  内部隐藏文件.是一个  内部的目录内部的..是一个  
在目录内每创建一个目录都会给目录增加一个硬链接数目。

###使用硬链接的意义  
防止重要文件被误删  
若有多人需要使用同一个文件，则可以创建多个硬链接
（比较大的文件在服务器中可以只存储一分，多个使用者）

##3、 两者的区别
不同点：  
软连接是快捷方式，用在硬盘空间的合理分配上。
硬链接是文件的名称，一个文件的多个名称互为硬链接。

相同点：  
都是针对同一个目标进行操作
